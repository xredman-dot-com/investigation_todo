# 格物清单 实现计划 v2（TickTick 对齐 + 格物路线）

**版本**: v2.0
**日期**: 2026-02-13
**制定者**: Claude + User

---

## 目标与范围
- **V1 目标**：个人版与 TickTick 核心功能对齐（不做团队协作）。
- **V2 目标**：以“反思/回顾”作为差异化核心能力。
- **技术栈**：沿用既定三端架构与技术选型（不变更）。

## 约束与前置
- 微信订阅消息模板需提前准备与审核。
- 小程序端需保证离线场景可用（本地缓存 + 同步策略）。
- 以“可交付里程碑”拆解，允许 V1 内部阶段性发布。

## 关键产品决策（已确认）
- **视图优先级**：V1 先做时间线视图，Kanban 视图放入 V1.1。
- **提醒策略**：多提醒（提前 5/15/60/1440 分钟）。
- **同步冲突**：字段级合并（标题/日期分离）作为默认策略。
- **格物数据采集**：V1 不采集反思数据，仅预留字段。
- **习惯/番茄钟**：V1 必须同时具备。

## 里程碑拆解（V1）

### M0 - 基础设施与工程搭建
- **目标**：完成 Monorepo、基础目录、开发环境与基础依赖。
- **参考**：`docs/plans/2025-02-13-phase1-infrastructure.md`
- **验收**：三端可启动、基础依赖与配置齐备。

### M1 - 账号与基础任务域
- **后端**：用户体系（微信登录、Token）、清单/任务基础模型、CRUD API。
- **小程序**：登录态、任务列表页、任务详情/编辑、清单切换。
- **后台**：用户列表与基础详情。
- **验收**：用户可完成“创建清单/任务 -> 编辑 -> 完成/删除”。

#### M1 详细拆解（按端）
**后端**
- 设计并实现微信登录流程（code -> openid），用户创建/更新与 Token 颁发。
- 清单/任务基础模型与迁移（list/task 表与基础字段）。
- 基础 CRUD API：列表、任务、任务状态更新、排序字段更新。
- 基础鉴权与权限边界（用户数据隔离）。

**小程序**
- 登录态管理（wx.login + token 存储与刷新）。
- 任务列表页与清单切换（默认收件箱）。
- 任务详情/编辑页（标题、描述、日期/时间、状态）。
- 本地缓存与首屏加速（最近清单与任务）。

**后台管理**
- 用户列表与详情（分页/搜索）。
- 封禁/解封基础能力（可延后到 M2）。

**验收用例**
- 新用户首次登录自动创建默认清单。
- 任务基本字段可创建、编辑、完成、删除。
- 退出/重登后数据一致。

### M2 - 高级任务能力
- **后端**：重复规则、优先级、标签、子任务、附件上传、搜索。
- **小程序**：快速添加、标签/优先级/重复设置、附件预览。
- **后台**：任务检索与简单审计视图。
- **验收**：复杂任务可完整配置并正确展示。

#### M2 详细拆解（按端）
**后端**
- 重复规则解析与生成（完成后生成下一次任务）。
- 优先级、标签、子任务、附件字段与迁移。
- 附件上传（本地存储适配器，支持切换 COS）。
- 基础搜索（标题/描述/标签）与排序能力。
- 字段级合并策略（标题/日期冲突的合并规则）。

**小程序**
- 快速添加（支持自然语言解析）。
- 标签/优先级/重复/子任务编辑。
- 附件上传与预览。

**后台管理**
- 任务检索与简单审计视图（可按用户/时间筛选）。

**验收用例**
- 重复任务按规则生成下一次实例。
- 标签/优先级/子任务正确展示与编辑。
- 附件上传可预览并可删除。

### M3 - 视图与组织
- **后端**：筛选/智能清单、四象限、时间线接口。
- **小程序**：日历视图（月/周/日/列表）、时间线、四象限、筛选。
- **后台**：基础统计看板（任务完成率/活跃度）。
- **验收**：用户可在多视图间切换、过滤与组织任务。

### M4 - 提醒与通知
- **后端**：提醒规则、多提醒、延后、消息投递、订阅消息记录。
- **小程序**：订阅引导、提醒设置、到期提醒触达。
- **后台**：提醒投递记录、模板管理。
- **验收**：提醒按时触发，订阅消息闭环可用。

### M5 - 专注/习惯/统计
- **后端**：番茄钟、习惯打卡、专注统计、习惯统计。
- **小程序**：番茄钟、习惯页、统计报表。
- **后台**：全局统计概览。
- **验收**：专注/习惯数据可追踪，统计可视化。

### M6 - 小组件与完善
- **后端**：小组件数据聚合、导出/备份接口。
- **小程序**：任务/习惯/专注/统计小组件。
- **后台**：配置与运营入口（公告、维护模式）。
- **验收**：小组件稳定显示，基础运营能力可用。

## 里程碑拆解（V2 - 格物差异化）

### G1 - 反思与回顾
- **后端**：反思记录表、周/月回顾聚合、感悟索引。
- **小程序**：任务完成后的反思入口、日/周/月回顾。
- **后台**：回顾内容审计与运营配置。
- **验收**：反思与回顾可形成“任务 -> 感悟 -> 复盘”的闭环。

### G2 - 成长洞察
- **后端**：成长指标模型（完成质量、意义达成等）。
- **小程序**：洞察面板与成长趋势。
- **验收**：从“完成”升级为“成长”的反馈机制。

## 风险与注意点
- **微信订阅消息审核周期**影响提醒上线节奏。
- **多端同步冲突**需明确策略（最后写入/合并规则）。
- **离线与弱网**是小程序体验关键。
